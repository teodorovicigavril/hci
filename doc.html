1. The scope of application

This application connects friends and strangers together to enjoy and learn a different type of the well-known game Tic-Tac-Toe.

2. Introduction

We all know that mobile games are present in almost every smartphone devices. People enjoy playing games and what better way than a new game, a more evolved version of Tic-Tac-Toe game where players can try different strategies, play against computer or other players in order to achieve the highest places on game's top.

3. Application architecture

The application contains 9 pages.

The game starts with home page where the user have next options:

  • play the game
  • see rankings
  • access About page
  • go to Profile page

In order for someone to play the game and/or access profile page, he must login or register. Home page offers two buttons for this 'Login' and 'Register'. Anyone that have the game can access to Rankins and About Page. For a easier navigation, the AppBar contains a Menu Icon used for navigation through pages and logout functionality. From 'Profile Page', the user can go to 'Update Profile Page' where he can update his account details. When choosing to play the game, the user is sent to a 'Main Game Page' and then he can choose to play against computer or other players, after this the user is sent to the actual game page.

4. Application pages

4.1 Home Page

The Home Page is the page where user have different buttons linked to different pages. He can login/register and then start his experience.

4.2 Login Page

With this page the user can log in the application using the email and the password of the account. More options for logging are in progress.

4.3 Register Page

In case of a new user, or a user that wants a new account this is the page needed. Anyone in this situation should complete a formular with next information: email, password, username and profile images. This informations can be modified later on.

4.4 Profile Page

This page contains account informations about the current logged user. Here, the user can press 'Update profile' in order to complete a formular with the informations he need to change. Furthermore, here, the user can logout from app and see his status regading the winns and the win rate so far.

4.5 Update Profile Page

In order to access this page, you have to be logged in and to enter the current password. As explained before, here, the user complete the form in order to update his account informations.



Pop-up used for user when deleting or updating profile.

4.6 Rankings Page

On this page, users can see top 10 players from the last week and top players so far by win rate. The page also contains a search filter for every player that wants to know his ranking.

4.7 About Page

Here you can find informations about the game, how to play, how it was implemented, who implemented, contact informations and a video where you can see how to play the game.

4.8 Main Game Page

When pressing 'Play' button from 'Home Page', the user have to choose which mode, against computer or against other players (including friends - by code).

4.9 Game Page

This is the actual Game, here the player have the pieces and the table on the screen and play the game.

5.2 Design patterns

Next design patterns where used in the application:

  1. Input Feedback - This design pattern is used at login and registration process, if the password doesn't respect requirements of a safe password, the user will be warned properly.

  2. Input Prompt - Is present when the user completes forms of Login, Register or Feedback on About Page.

  3. Navigation Tabs - Is present on every page by a menu button which shows current page and possible pages to go.

  4. Home Link - The AppBar contains the logo of the app which is a direct link to the home page. Can be accesed from every page.

  5. Pagination - On 'Rankings' page a list of all users and their scores is present, in order to see all users you need to use pages.

  6. Continuous Scrolling - On 'Rankings' page a list of top 10 users and their scores is present.

  7. Leaderboards - Users want to know who are the very best performers in a category or overall.

  8. Clear Primary Actions - Actions like login/register, play game, give feedback, see rankings are present in the application being intuivive for the user.

6. Questions, Options, Criteria


Question 1: How can a user see his winning status?

 Option 1: He can see his status by tapping on Profile page.

 Option 2: The user can check his status by looking at Rankings page and search for his username.

 Criteria 1: Intuitive, efficiency.

 Criteria 2: By keyboard/mouse interaction. Useful when you want to see others scores and how much for a better place.


Question 2: How can a log-out?

 Option 1: The user can logout from navigation menu.

 Option 2: The user can logout from profile page.

 Criteria 1: Efficiency, can logout from any page, accesibility.

 Criteria 2: Intuitive.


Question 3: How can a user learn/see how to play the game?

 Option 1: By pressing on 'About' page, a video and some text describing the game is presented.

 Option 2: By pressing on 'Game' page for first time, a quick tutorial is shown with pop-ups.

 Criteria 1: Intuitive, watching, learing in own rhythm.

 Criteria 2: Efficiency, learning by doing, can be skiped.
